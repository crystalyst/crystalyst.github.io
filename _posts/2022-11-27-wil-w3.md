#### Things I did and learned

Node.js

JavaScript 엔진으로 빌드된 JavaScript 런타임 

원래는 웹 브라우저에서만 사용할 수 있었는데, 컴퓨터에서 사용할 수 있게 만들어 줌 (브라우저가 없어도 사용할 수 있도록 함)

Node.js -> 자바스크립트를 활용할 수 있는 환경 그 자체


- Non-blocking I/O
    - 프로그램 내부에서 함수를 호출하면 호출된 함수가 작업을 마칠때까지 기다렸음 (기존에 - blocking I/O)
    - Non-blocking I/O는 함수가 실행되는 와중에도 다른 작업을 동시에 진행할 수 있는 장점이 있음 (비동기 작업을 할 떄 더 효율적으로 사용이 가능함)
- Single Thread
    - 스레드 하나를 사용함 (동시에 하나의 작업만을 처리할 수 있음)
    - 스레드란? 프로그램이 동작할 떄 cpu 또는 프로세서를 사용하는 단위
        - 일반적으로 1개의 프로그램은 1개의 스레드를 사용할 수 있음. 만약 1개의 프로그램에서 여러개의 스레드를 사용할 수 있다면 연산 시간이 단축될 수 있음.
        - Node.js 의 이 부분은 단점이 될 수도 있는데, 병렬/멀티스레드 프로그래밍을 이해하지 않아도 무방한 장점도 있음
- Event loop
    - 싱글 스레드의 약점을 극복하기 위해 효율적으로 작업을 처리할 수 있는 특성

- ES6 (ECMAScript 6)
    - 각 브라우저마다 개발하는 방식/언어가 다르니 그들을 표준화하는 방식이 필요했고, 그래서 나온 것이 ECMAScript임.
    - ES6 === ECMAScript 의 6번째 버전 / 2015 (ES5와 ES6는 서로간 문법이 많이 다르다)

- REPL 환경
    - 입력받은 코드를 읽어 (Read) 들여 메모리에 저장하고, 평가(Evaluate)된 값을 출력 (print)함. 특정 신호를 입력받기 전까지 해당 과정을 반복 (loop)함
    - 입력하는 코드의 결과값을 즉시 확인할 수 있기에 간단한 문법 테스트 등에 매우 적합한 개발환경이라고 할 수 있음.
    - 웹브라우저 콘솔에서도 자바스크립트 코드를 작성/체크할 수 있으나 (repl) Node.js가 제공하는 API를 사용할 수 없음 (브라우저의 API를 사용함)
- 동기/비동기
    - 동기: 먼저 실행된 코드의 결과가 나올때까지 대기하는 것
        - 놀이기구 탑승/하차가 동기적으로 관리된다 - 놀이기구가 작동하고 있는 기간동안 사람들이 승하차할수는 없음
    - 비동기: 실행된 순서와 관계 없이 결과가 나오는 것 의미
        -  맛집의 경우 다 먹은 사람이 생길때마다 들어오고 나오는 등 활동이 일어남
- Blocking Model / Non-blocking model
    - Blocking model: 코드의 실행이 끝나기 전까지 실행 제어권을 다른 곳에 넘기지 않아 다른 작업을 하지 못하고 대기
    - Nonblocking model: 코드의 실행이 끝나지 않아도 실행 제어권을 다른 곳에 넘겨 다음 코드가 실행될 수 있음 (JavaScript = async + non-blocking model)
    - JavaScript -> 각 명령들이 순서대로 실행될 수 있게 구현되어 있음 / non-blocking model에 의해 명령이 아닌 모든 함수는 비동기적으로 실행됨.
- Promise
    - 자바스크립트에서 비동기 처리를 동기로 처리할 수 있게 돕는 객체 유형 -> 비동기 처리를 좀 더 손쉽게 할 수 있음
    - Promise Constructor Interface
        - Promise(executor) -> executor에는 함수만 올 수 있으며 인자로 resolve, reject가 주입됨
        - Executor: 실행 함수 / Promise가 만들어질 때 자동으로 실행됨
        - Promise가 연산을 언제 종료하는지 상관하지 않고, resolve/reject 중 하나를 무조건 호출해야 함.
        - Promise 연산 중 에러가 발생할 시 reject를 호출하게 됨
    - Constructor?
        - JavaScript에서는 Primitive Type등을 제외한 타입들이 대부분 객체 (object)로 되어 있음
        - 일반적으로 객체를 생성하는 함수를 constructor라고 부름 -> promise 도한 객체로 구성되어 있기에 constructor 사용.
    - Promise의 상태
        - 반드시 3개의 상태를 가지며, 대기(Pending)상태가 아니라면 연산이 이미 끝난 상태.
            - Pending: 이행하거나 거부되지 않은 초기 상태 (연산/코드가 실행되는 단계)
            - Fulfilled: 연산이 성공적으로 완료됨
            - Rejected: 연산이 실패함
        - Promise가 만들어질 때 executor가 실행되며, executor에서 resolve가 호출되지 전까지는 firstPromise.then()이 실행되지 않는다.
        - resolve가 실행이 된 경우 fulfilled 상태가 됨
        - resolve()가 fulfill 되었을때 실행되는 프로미스 -> fulfilled promise (.then()을 통하여 코드가 실행됨)
    - promise.then
        - Promise 안에서 resolve가 실행된 경우 then method에 작성된 함수가 실행됨
    - promise.catch
        - Reject가 실행됨 경우 catch 메소드에 작성된 함수가 실행됨
    - 함수형 프로그래밍 체험
        - promise.then(console.log) -> Promise.resolve() 에 넘겨준 argument를 출력해주는 로직 (함수를 그대로 then으로 넘겼기에 실행이 되면서 argument를 자동으로 console.log에 할당함.
    - promise.then(increment).then(increment)… 3 나오는 점에 대해 제대로 정리해보기
- 비동기 함수 (async function)
    - 비동기 함수의 결과값은 항상 promise object로 resolve 됨
    - 비동기 함수 안에서만 await 연산자를 사용할 수 있음.
    - 기존처럼 일반 함수나 화살표 함수처럼 사용하는 것이 가능함.
